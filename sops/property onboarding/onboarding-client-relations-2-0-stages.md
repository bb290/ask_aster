:::info
Every onboarding moves through 4 stages in the Client Relations 2.0 Asana project. These map to the Asana project sections.

:::

## Task Templates

* `(CLIENT) <Address> / <Owner Name>` — new client (see [Onboarding Asana SOP](../../docs/section-indexes/sops-2-onboarding-asana-sop.md))
* `(PROPERTY) <Address> / <Owner Name>` — existing client adding a property (see [Adding Property variant](../../docs/section-indexes/sops-2-onboarding-asana-sop-adding-property.md))
* `(UNIT) <Address> / <Owner Name>` — existing client adding a unit (see [Adding Unit variant](../../docs/section-indexes/sops-2-onboarding-asana-sop-adding-unit.md))

## Stages


1. **Initial Onboarding** — Account setup, data entry, initial owner contribution, utility transfer, lease-up or PM transfer (or both), high-priority issues if applicable (rent collection, eviction, deferred or major maintenance).
2. **Messy Middle** — Transitional time to stabilize property and complete the transfer. Review every Tuesday and Friday. See [Onboarding // Messy Middle](onboarding-messy-middle.md).
3. **Finalize Onboarding** — Initial owner distribution + reporting set, optional review call with Accounting Manager, vendor transfer + setup, final data-entry double check, feedback request, KPI tracking.
4. **Active** — Once onboarding is finalized, the task moves to Active. All active properties live here until off-boarding.
