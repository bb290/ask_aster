> **Situation:** Existing client wants to add a unit (ADU, DADU, conversion, or reclaimed unit) to a property we already manage.


---

## Key Points

* **Existing property and owner** — most onboarding steps are skipped; this uses the [Adding Unit variant](../../docs/section-indexes/sops-2-onboarding-asana-sop-adding-unit.md), not a full onboarding.
* **PMA must cover the unit** — the existing PM Agreement's address may be unit-specific (e.g. `5521 Brooklyn Ave #A`); if so, a PMA revision is required before the new unit can be added.
* **Info form drives the rest** — send the owner the [Adding Unit Form](https://form.jotform.com/260167444758162) to capture unit details. Everything else is internal.
* **No onboarding fee** — existing clients are not charged an onboarding fee when adding a unit.
* **Convert to subtask** — the "Adding Unit" task becomes a subtask of the existing Client Relations task for that property.

## Procedure

Follow the [Adding Unit variant](../../docs/section-indexes/sops-2-onboarding-asana-sop-adding-unit.md) in the Client Relations 2.0 project:


1. **VERIFY | Existing PMA includes all units** — confirm the PMA address is general, not unit-specific; revise via PandaDoc if needed.
2. **CONVERT | To Subtask of existing Client Relations task** — move the "Adding Unit" task under the property's Client Relations parent task.
3. **SEND | Adding Unit Detail Confirmation Email** — the template above.
4. **ADD | Unit to Buildium** — Buildium → Property → Units → Add Unit.
5. **UPDATE | Unit Count on Mgmt Dashboard** — bump `# of Units` for the existing address.
6. **UPDATE | Google Map** — edit the unit count on the [Sagareus AUM Google Map](https://www.google.com/maps/d/u/0/edit?mid=1nVYVXbnQqzOlV17EFVaY1skcj7Gg98w&ll=47.52830654532093%2C-122.4681203&z=9); if the property is converting from single-family to multi-family, move the row between data tables.
7. **SEND | Onboarding Complete (Adding Unit)** — confirmation + [1-question survey](https://survey.hsforms.com/1x4HwRg4jQz2Na8dyFE-IhQ3n3v9).
