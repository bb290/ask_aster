> **Situation:** Existing client wants to add a new property to their portfolio under the existing management relationship.


---

## Key Points

* **Existing client, streamlined flow** — uses the [Adding Property variant](../../docs/section-indexes/sops-2-onboarding-asana-sop-adding-property.md); skips EFT enablement, onboarding fee, and portal welcome email.
* **New PM Agreement addendum required** — a PMA addendum signs the new property under the existing management relationship before system setup begins.
* **Standard Onboarding Form** — the full Initial Onboarding Form is used (not the Accounting-Only variant), since we still need property-level data.
* **Same SLAs apply** — 2 days to Initial Onboard Complete, 21 days to Initial Reports Sent, 60 days to Onboarding Finalized.
* **Pick existing owner in Buildium** — when adding the property in Buildium, select the existing Rental Owner rather than creating a new profile.

## Procedure

Follow the [Adding Property variant](../../docs/section-indexes/sops-2-onboarding-asana-sop-adding-property.md) / [Onboarding - New Client](/doc/onboarding-new-client-oDcEnhe9Qv)<!-- unresolved: doc not migrated --> lifecycle, skipping these three steps:

* `UPDATE | EFT Payments` — owner already has EFT enabled
* `ADD | Onboarding Fee` — waived for existing clients
* `SEND | Owner Portal Welcome email` — owner already has portal access

**Key differences:**


1. **SEND | PM Agreement Addendum** — PandaDoc → Templates → BizDev → Addendum to Property Management Agreement → add the new property address.
2. **ADD | Property to Buildium** — choose the **existing** Rental Owner rather than creating a new one.
3. **SEND | Finalize Onboarding email** — use the existing-client variant that confirms ownership & banking details are unchanged. If details differ for this property, have the client complete a separate Accounting Only Onboarding Form.
