> **Situation:** A vendor submits an estimate (via QuickBooks, email, or other method) for property work such as roof cleaning, gutter cleaning, repairs, or inspections. Management must route the estimate to the appropriate person for review and approval.


---

## Key Points

* **Search Asana for an existing task** — Look up the property address in Asana to determine if someone already requested this estimate. Common places to look: Ops Team, Maintenance 2.0, Annual Inspection, Multifamily Inspection.
* **IF a task exists — attach and notify the requestor:**
  * Attach the estimate (or link to the estimate) to the existing Asana task
  * Update the task due date to today so the assignee sees it promptly
  * Forward the original estimate email to the person who requested the estimate
* **IF no task exists — create a new task:**
  * Create a new task in the **Ops Team** project in Asana
  * Title: `Vendor Estimate // [Vendor Name] // [Property Address]`
  * Assign to: Ops Manager
  * Due: Today
  * Attach or link the estimate in the task
  * Add a comment with context (vendor name, scope of work, property)


---

## Procedure


1. Receive the vendor estimate via email or QuickBooks notification
2. Identify the property and vendor from the estimate
3. Search Asana (Ops Team, Maintenance 2.0, Annual Inspection, Multifamily Inspection) for an existing task related to this estimate
4. **If task found:** Attach the estimate to the task, update due date to today, and forward the estimate email to the task assignee
5. **If no task found:** Create a new Asana task in Ops Team with the estimate attached, assigned to the Ops Manager, due today
6. Reply to the vendor confirming receipt (if the estimate came via direct email)


---

### Example Vendor Questions

* "Here is the estimate for the roof cleaning at 1234 Main St."

* "We've completed the inspection and attached our estimate for the needed repairs."
* "Please find the estimate for the work discussed at \[Property Address\]."
