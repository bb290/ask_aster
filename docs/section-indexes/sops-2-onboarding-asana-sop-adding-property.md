:::info
This variant covers **existing clients adding a property** to their portfolio. It follows the [Onboarding Asana SOP](sops-2-onboarding-asana-sop.md) lifecycle, with 3 steps skipped.

:::

## When This Runs

An existing Sagareus client is adding a new property to their portfolio (they already have an owner profile and EFT enabled). Use the **(PROPERTY) <Address> / <Owner Name>** task template in Client Relations 2.0.


---

## Skipped Steps

These 3 steps do NOT apply to Adding Property onboardings — the existing client is already set up:

* [**Step 9 — UPDATE | EFT Payments**](../../sops/property onboarding/9-update-eft-payments.md) — owner already has EFT enabled
* [**Step 10 — ADD | Onboarding Fee**](../../sops/property onboarding/10-add-onboarding-fee.md) — onboarding fee is waived for existing clients
* [**Step 11 — SEND | Owner Portal Welcome email**](../../sops/property onboarding/11-send-owner-portal-welcome-email-from-buildium.md) — owner already has portal access


---

## Key Differences

### Step 3 — ADD | Property + Client to Buildium

**Choose the existing owner** rather than creating a new one. All other fields on the Add Property form are entered the same as in the New Client flow — see [step 3](../../sops/property onboarding/3-add-property-client-to-buildium.md) for the field list.

### Step 23 — SEND | Finalize Onboarding email

Use the **existing client variant** of the Finalize Onboarding email, which confirms that ownership and banking details are unchanged. If details have changed, have the client complete a separate Accounting Only Onboarding Form.
