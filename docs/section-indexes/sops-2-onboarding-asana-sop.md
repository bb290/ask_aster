:::info
This SOP mirrors the **(CLIENT)** task template in the Client Relations 2.0 Asana project. Each numbered child corresponds to a subtask in the template, in order. Work the subtasks; open the linked step doc for detail.

:::

## How to use


1. Open the Onboarding parent task in Asana (Client Relations 2.0 → Onboarding section)
2. Work each subtask in order
3. Click into the step doc for the detailed procedure
4. Mark the subtask complete when finished

## Variants

* [**Onboarding Asana SOP // Adding Property**](sops-2-onboarding-asana-sop-adding-property.md) — existing client adding a property. Same lifecycle as below, minus steps 9, 10, 11.
* [**Onboarding Asana SOP // Adding Unit**](sops-2-onboarding-asana-sop-adding-unit.md) — existing client adding a unit to an existing property (ADU, DADU, conversion). Shorter mini template.

## Stage Reference

| Stage | Steps | SLA |
|-------|-------|-----|
| **Initial Onboarding** | 1 – 19 | 2 days to Initial Onboard Complete |
| **Messy Middle** | 20 – 22 | 21 days to Initial Reports Sent |
| **Finalize Onboarding** | 23 – 29 | 60 days to Onboarding Finalized |

See [Onboarding // Dashboard and Metrics](../../sops/property onboarding/onboarding-dashboard-and-metrics.md) for KPI definitions and the [analytics dashboard](https://d31czylo2z0inj.cloudfront.net/#onboarding).
